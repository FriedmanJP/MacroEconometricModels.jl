Downloading IO/MRIO Data

The module can download several public multi-regional IO (MRIO) databases, modeled on the Python pymrio toolkit. Downloading only fetches the files; converting an archive into an IOData is done separately with parse_io (which needs the optional ZipFile/XLSX packages for zipped or Excel sources).

Listing sources

using MacroEconometricModels
list_io_sources()
Available IO/MRIO sources:
  :eora26 — EORA26 (credentials required)
      versions: 26
      Requires worldmrio.com account (email/password).
  :exiobase3 — EXIOBASE 3
      versions: 3.8.2
      Zenodo-hosted; record page scraped for IOT_YYYY_{pxp,ixi}.zip.
  :gloria — GLORIA
      versions: 053
      Global Resource Input-Output Assessment; fixed URL set.
  :oecd — OECD ICIO
      versions: v2016, v2018, v2021, v2023
      Inter-Country Input-Output tables; fixed per-version URLs.
  :wiod — WIOD 2013
      versions: 2013
      World Input-Output Database; HTML scrape for wiot*.xlsx.

The supported sources are OECD ICIO, WIOD 2013, EXIOBASE 3, EORA26 (login required), and GLORIA.

Downloading

download_io dispatches by source symbol and returns an IOMetaData log recording every url => file it fetched. Per-source wrappers are also exported.

# Download the latest OECD ICIO tables (2016–2020 block) into ./mrio
meta = download_io(:oecd; storage_folder="mrio", version="v2023")

# only specific years
download_oecd("mrio"; version="v2016", years=[2000, 2001])

# EXIOBASE 3 (product-by-product), all years on the Zenodo record
download_exiobase3("mrio"; system="pxp")

# EORA26 needs an account
download_eora26("mrio"; email="you@example.com", password="…")

Downloads skip files that already exist unless overwrite_existing=true.

Parsing archives into IOData

parse_io detects the file type and dispatches. CSV/TSV files parse in-core; .zip and .xlsx require the relevant package to be loaded:

using ZipFile, XLSX        # enable the parser extensions

io = parse_io("mrio/ICIO_v2023_2016-2020.zip"; source=:oecd, member="…csv",
              n_sectors=45, n_fd=1)

If the required package is not loaded, parse_io raises an actionable error telling you to ]add ZipFile or ]add XLSX.

MacroEconometricModels.parse_io — Function
parse_io(path; source, year=nothing, kwargs...) -> IOData

Parse a downloaded IO file into an IOData. Dispatches on the file extension: .csv/.tsv/.txt are parsed in-core (via DelimitedFiles), while .zip and .xlsx require the optional ZipFile / XLSX packages (loaded as package extensions) and raise an actionable error if those are not available.

MacroEconometricModels.IOMetaData — Type
IOMetaData

Provenance / download log for an IO table (analogue of pymrio's MRIOMetaData). files maps each source url => local filename; history is a list of timestamped log lines.
